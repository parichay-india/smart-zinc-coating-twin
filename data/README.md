# Data folder

Place the plant process logs here:

- `PeriodA.csv` — legacy regime (open-loop, operator judgement)
- `PeriodB.csv` — deployed regime (closed-loop on PG-HGBT)

Each file is a 5-minute-cadence process log with the columns used by the model
(`processSpeed, width, thickness, frontPressure, rearPressure, frontHP, rearHP, vPos,
frontTilt, rearTilt, potTemp, snoutTemp, frontCoating, lineState, overCoat_pct,
zincConsumed_kg, targetZinc_kg, excessZinc_kg`, …).

**These CSVs are intentionally git-ignored** (they are large and may be proprietary). The
repository ships a pre-trained model (`models/pg_hgbt_frontcoating.pkl`) and the validated
operational KPIs (`kpis.json`), so the app runs without the raw CSVs. Add the CSVs only if
you want to **retrain** (`python train_models.py`) or recompute the before/after KPIs.

> Provenance note: if the deployed-period log is generated by the calibrated digital twin
> rather than measured on the line, label it as such in any external report — the KPIs in
> `kpis.json` should then be described as twin-projected, not measured.
